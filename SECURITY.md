# 安全说明

## 公开提交前检查

请勿在Issue、Pull Request或配置文件中提交：

- 节点订阅URL；
- 节点地址、用户名、密码；
- API Key、Token、Cookie、二维码；
- 个人公网IP、设备日志或账号信息。

## 报告问题

先查看仓库的 **Security** 页面：

- 如果能看到 **Report a vulnerability**，请使用该入口私密提交；
- 如果看不到该入口，不要用公开 Issue 或 Pull Request 提交敏感内容，也不要附带可复用凭证或完整日志。请等待维护者启用私密报告入口，或只公开描述不包含任何敏感数据的一般现象。

私密漏洞报告是 GitHub 仓库的远端设置，不会因为合并本文件而自动开启。

## 能力边界

本项目仅提供流量分流规则，不提供账号风控规避、设备指纹伪装、IP信誉保证或匿名承诺。
