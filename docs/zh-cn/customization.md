# 自定义指南

## 推荐原则

1. 先专项、后通用；
2. 先域名、后GEOIP；
3. 国内服务DIRECT，海外服务PROXY；
4. 不把节点、订阅和规则混在同一个公开文件；
5. 不为了追求规则数量而增加不稳定依赖。

## 添加海外服务

在“AI”或“开发与指定海外服务”区域加入：

```ini
DOMAIN-SUFFIX,example.com,PROXY
```

需要远程解析时可以使用经过验证的远程规则集。

## 添加国内服务

在中国大陆通用规则之前加入：

```ini
DOMAIN-SUFFIX,example.cn,DIRECT
```

不要把国内专项规则放到`FINAL,PROXY`之后。

## 添加广告规则

仓库默认关闭广告拦截，避免误伤办公登录、支付和游戏奖励广告。如需开启，应自行评估误拦截风险。

## 节点与订阅

Shadowrocket配置的`[Proxy]`和`[Proxy Group]`保持为空。`PROXY`策略会跟随首页当前手动选择的节点。

不要把以下内容提交到公开仓库：

- 订阅URL；
- 节点服务器地址；
- 用户名和密码；
- Token、Cookie、二维码或可复用登录信息。

## 更新策略

公开仓库的Raw URL适合分发稳定版本。若需要长期自定义，建议Fork后修改自己的版本，避免上游更新覆盖本地改动。
