# 排查指南

## 启用后完全断网

1. 立即切回原配置或关闭Shadowrocket；
2. 确认订阅已经更新；
3. 确认首页已手动选择可用节点；
4. 确认全局路由为“配置”；
5. 检查远程规则URL是否可访问。

## 配置没有自动更新

1. 优先确认配置是通过GitHub Raw远程URL导入；若使用恢复文件导入，确认纯文本中的`update-url`仍然存在；
2. 在“设置 → 订阅”开启“配置自动后台更新”和“更新通知”；
3. 将更新间隔设为3天；
4. 在系统设置中允许Shadowrocket使用“后台App刷新”；
5. 不要强制退出Shadowrocket，系统重启后主动打开一次；
6. 仍有疑问时，手动更新远程配置并核对配置版本。

手动路径：在“配置”列表点击配置文件名称 → “更新” → 再次点击文件名称 → “预览”。更新入口不在右侧信息符号中。

iOS和macOS会自行调度后台任务，因此无法保证在某个固定时刻立即更新。机场订阅的节点更新也需要单独启用，不会被本配置覆盖。

## 国内App变慢

1. 检查目标域名是否命中DIRECT；
2. 确认“包括所有网络”和“强制路由”没有开启；
3. 排除当前Wi-Fi、5G或运营商波动；
4. 不要只依据一次测速判断配置。

## 海外服务打不开

1. 在首页手动更换节点；
2. 检查节点是否支持目标服务和地区；
3. 分别测试ChatGPT、Google和GitHub；
4. 401/403只代表端点可达但请求未获授权或被拦截；
5. 不要把账号风控直接归因于分流配置。

## 局域网设备不可用

1. 确认“包括所有网络”关闭；
2. 确认“包括本地网络”关闭；
3. 检查设备是否位于常见私有地址范围；
4. 临时关闭Shadowrocket验证是否确实由隧道引起。

## 如何报告问题

可以提供：

- 设备类型与系统版本；
- Shadowrocket版本；
- 出问题的公开域名或App名称；
- 预期DIRECT还是PROXY；
- 是否发生在Wi-Fi或蜂窝网络；
- 已脱敏的错误类型和时间。

不要公开提交订阅URL、节点、账号、Cookie、公网IP、二维码或完整日志。
