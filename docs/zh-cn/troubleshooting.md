# 排查指南

## 启用后完全断网

1. 立即切回原配置或关闭Shadowrocket；
2. 确认订阅已经更新；
3. 确认首页已手动选择可用节点；
4. 确认全局路由为“配置”；
5. 检查远程规则URL是否可访问。

## 国内App变慢

1. 检查目标域名是否命中DIRECT；
2. 确认“包括所有网络”和“强制路由”没有开启；
3. 排除当前Wi-Fi、5G或运营商波动；
4. 不要只依据一次测速判断配置。

## 海外服务打不开

1. 在首页手动更换节点；
2. 检查节点是否支持目标服务和地区；
3. 分别测试ChatGPT、Google和GitHub；
4. 401/403只代表端点可达但请求未获授权或被拦截；
5. 不要把账号风控直接归因于分流配置。

## 局域网设备不可用

1. 确认“包括所有网络”关闭；
2. 确认“包括本地网络”关闭；
3. 检查设备是否位于常见私有地址范围；
4. 临时关闭Shadowrocket验证是否确实由隧道引起。

## 如何报告问题

可以提供：

- 设备类型与系统版本；
- Shadowrocket版本；
- 出问题的公开域名或App名称；
- 预期DIRECT还是PROXY；
- 是否发生在Wi-Fi或蜂窝网络；
- 已脱敏的错误类型和时间。

不要公开提交订阅URL、节点、账号、Cookie、公网IP、二维码或完整日志。
